# 中间件（Middleware）

# 概述

中间件实际上是在请求到达路由方法之前的执行函数，它可以获取到request及response的上下文，以及next()方法调用下一个中间件。

Nest.js默认情况下和express的中间件是保持一致的机制。

# 示例

import { Injectable, NestMiddleware } from "@nestjs/common";
import { Request, Response } from "express";

@Injectable()
export class LoggerMiddleware implements NestMiddleware {
  use(req: Request, res: Response, next: Function) {
    console.log("Request...");
    next();
  }
}

实现Nest中间件，需要实现NestMiddleware接口，并且使用Injectable装饰器。

# 函数中间件

export function logger(req, res, next) {
  console.log(`Request...`);
  next();
}

函数中间件约束较少，主要为了和express生态兼容，一般而言使用类的方式会更好一些。

# 模块内应用中间件

import { Module, NestModule, MiddlewareConsumer } from "@nestjs/common";
import { LoggerMiddleware } from "./common/middleware/logger.middleware";
import { CatsModule } from "./cats/cats.module";

@Module({
  imports: [CatsModule],
})
export class AppModule implements NestModule {
  configure(consumer: MiddlewareConsumer) {
    consumer.apply(LoggerMiddleware).forRoutes("cats");
  }
}

在模块内部使用中间件，需要实现NestModule接口，利用MiddlewareConsumer这个 helper 去注册中间件（支持多个参数），forRoutes方法指定应用的路由。

middlewareConsumer是遵循链式调用的，其中forRoutes方法可以接受路由类本身：

import { Module, NestModule, MiddlewareConsumer } from "@nestjs/common";
import { LoggerMiddleware } from "./common/middleware/logger.middleware";
import { CatsModule } from "./cats/cats.module";
import { CatsController } from "./cats/cats.controller.ts";

@Module({
  imports: [CatsModule],
})
export class AppModule implements NestModule {
  configure(consumer: MiddlewareConsumer) {
    consumer.apply(LoggerMiddleware).forRoutes(CatsController);
  }
}

# 全局中间件

如果想作用于所有路由上，可以使用全局中间件，更加简单：

const app = await NestFactory.create(AppModule);
app.use(logger);
await app.listen(3000);
