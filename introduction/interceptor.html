# 拦截器（Interceptor）

# 概述

拦截器在Nest.js中，实现了NestInterceptor，是遵循AOP思想的：

import {
  Injectable,
  NestInterceptor,
  ExecutionContext,
  CallHandler,
} from "@nestjs/common";
import { Observable } from "rxjs";
import { tap } from "rxjs/operators";

@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    console.log("Before...");

    const now = Date.now();
    return next
      .handle()
      .pipe(tap(() => console.log(`After... ${Date.now() - now}ms`)));
  }
}

@UseInterceptors(LoggingInterceptor)
export class CatsController {}

或者在全局使用：

const app = await NestFactory.create(AppModule);
app.useGlobalInterceptors(new LoggingInterceptor());

# 关于 AOP 编程

AOP 是Aspect Oriented Programming的缩写，即面向切面编程，它是设计模式的一种延续，让调用者和被调用者充分解耦，提高灵活性和可扩展性。

一些常见的功能如日志记录、性能统计、安全处理、事务处理、异常处理，这些都与具体的业务逻辑无关，它们均可以从业务逻辑里抽离出来，通过业务逻辑切口进行一个弱耦合，达到低耦合性。
