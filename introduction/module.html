# 模块（Module）

# 概述

模块是Nest.js组织应用架构的方式，在表现上，使用@Module()装饰器修饰类。

import { Module } from "@nestjs/common";
import { CatsController } from "./cats.controller";
import { CatsService } from "./cats.service";

@Module({
  controllers: [CatsController],
  providers: [CatsService],
})
export class CatsModule {}

# 参数说明

  • providers，可被依赖注入的 provider，模块中的组件都可以使用
  • controllers，路由集合
  • imports，如果要使用其他模块的 providers，需要将其他模块加入到 imports 中来
  • exports，导出自己的 provider，可供其他模块使用

模块本身是自带封装性的，如果Provider没有被加到模块中，是无法独立被依赖使用的。

# 模块分类

# 根模块（Root Module）

每个应用都需要有一个入口模块，即根模块（Root Module），在Nest.js启动后，会根据当前的根模块构建一个有向图，查找依赖关系，逐层执行。

# 特性模块（Feature Module）

将特性相关的代码（Controller、Service）组织在一起，有助于我们建立起明显的边界，同时有助于我们实现符合SOLID原则的代码。

import { Module } from "@nestjs/common";
import { CatsController } from "./cats.controller";
import { CatsService } from "./cats.service";

@Module({
  controllers: [CatsController],
  providers: [CatsService],
})
export class CatsModule {}

import { Module } from "@nestjs/common";
import { CatsModule } from "./cats/cats.module";

@Module({
  imports: [CatsModule],
})
export class AppModule {}

# 共享模块（Shared Module）

在Nest.js中，任何一个带有export的Module都可以称作共享模块，在其他模块可以通过import的方式，使用模块导出的Service：

import { Module } from "@nestjs/common";
import { CatsController } from "./cats.controller";
import { CatsService } from "./cats.service";

@Module({
  controllers: [CatsController],
  providers: [CatsService],
  exports: [CatsService],
})
export class CatsModule {}

# 全局模块（Global Module）

如果一个Module被很多模块所import，属于比较通用的Module，则可以考虑注册为全局模块，减少import的重复：

import { Module, Global } from "@nestjs/common";
import { CatsController } from "./cats.controller";
import { CatsService } from "./cats.service";

@Global()
@Module({
  controllers: [CatsController],
  providers: [CatsService],
  exports: [CatsService],
})
export class CatsModule {}

此时在任何地方都可以注入使用CatsService，请注意不要滥用全局模块的方式。
