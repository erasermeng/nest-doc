# 准备工作

安装Node.js(>=10.13.0)

# 创建一个项目

npm i -g @nestjs/cli
nest new my-project

# 源码目录说明

. src
├── app.controller.spec.ts 控制器测试文件
├── app.controller.ts 主应用控制器文件（路由）
├── app.module.ts 主应用模块文件
├── app.service.ts 主应用服务文件
└── main.ts 主入口文件

# main 入口详解

import { NestFactory } from "@nestjs/core";
import { AppModule } from "./app.module";

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(3000);
}
bootstrap();

可以看到 main 入口文件中，调用了NestFactory.create方法，创建了一个 app 实例，该实例默认为一个express的实例（可通过传入 httpAdapter 替换），调用该实例的 listen 方法监听服务端口。

# module 详解

在main入口中，我们创建 Nest 实例时传入了AppMoudule，Nest 在启动时会根据该 Module 分析出相关的依赖，逐层加载相应的 Controller、Provider 等，完成应用的启动。

import { Module } from '@nestjs/common';
import { AppController } from './app.controller';
import { AppService } from './app.service';

@Module({
  imports: [],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {

# Service 详解

import { Injectable } from "@nestjs/common";

@Injectable()
export class AppService {
  getHello(): string {
    return "Hello World!";
  }
}

Service文件中定义了一个Provider，使用Injectable装饰器，让它的实例可以被注入到其他的场景（如Controller）中使用。

# Controller 详解

import { Controller, Get } from "@nestjs/common";
import { AppService } from "./app.service";

@Controller()
export class AppController {
  constructor(private readonly appService: AppService) {}

  @Get()
  getHello(): string {
    return this.appService.getHello();
  }
}

这是一个非常简单的 Controller，定义了一个/的路由，返回 Service 的getHello()方法结果。
